写在前面

Cloudify是一个混合云编排工具，用的人很少（毕竟大多数人都没有相关需求），Cloudify-agent用的人就更少了orz 我很犹豫要不要把这篇放到博客上面（毕竟很可能完全没人对这篇有兴趣orz）。最后决定放上来，也正是因为用的人少，资料更少，甚至官方都没有这方面的文档。希望我这篇东西能在某天为正好有此需要的你提供一点点便利。

这两天为了给同事提供一份在目标虚拟机上手动安装cloudify-agent的文档，我在cloudify-agent上面花了超多时间，打了n多次断点，眼睛都快看瞎了，还好终于搞清楚它的运转流程了（吐槽一句，这部分代码实现有点乱，看的心累orz，怪不得cloudify官方最近再重构）。这篇博客将记录手动安装cloudify-agent的方法，如果你碰巧也再搞这个（甚至就是搜索进来的），希望能帮你节约到一些时间和精力。

Cloudify-agent的本质就是配置机器监听特定的消息队列，不管Linux还是Windows的cloudify-agent，跑的都是Python脚本（在Windows下面，安装包会自带一套Python环境），下面会以Linux为例将一些步骤的要点，Windows下面就不在赘述了orz

Linux下的Cloudify-agent手动安装

1.配置环境

1

"/opt/cloudify_agent/deamon_name/env"/bin/python "/opt/cloudify_agent/deamon_name/env"/bin/cfy-agent configure --relocated-env --disable-requiretty

作用：

  • 设置sudo免密
  • 设置正确的环境变量和脚本地址

2.创建代理

1

"/opt/cloudify_agent/deamon_name/env"/bin/python "/opt/cloudify_agent/deamon_name/env"/bin/cfy-agent daemons create --broker-get-settings-from-manager --name=deamon_name --broker-ip="mq_ip" --broker-user="cloudify" --broker-pass="mq_password" --rest-host="mq_ip" --process-management="init.d" --bypass-maintenance_mode=false --min-workers=1 --workdir="/opt/cloudify_agent/deamon_name/work" --local-rest-cert-file="/opt/cloudify_agent/deamon_name/cloudify/ssl/cloudify_internal_cert.pem" --rest-port=53333 --max-workers=2 --user="root" --broker-ssl-cert-path="/opt/cloudify_agent/deamon_name/cloudify/ssl/cloudify_internal_cert.pem"  --queue="deamon_name" --rest-tenant=“default_tenant"

需要注意的参数：

  • host、deployment_id这些参数只有在queue没传的时候才生效，为了就是重新获取queue名称, 如果是手动安装，所有跟rest相关的都可以不加
  • 整个daemon是类似单例模式，是靠保存一个文件来实现单例的，name是唯一标识，手动安装如果无法删掉daemon的话，name一定会重复，到时肯定报错了，文件保持在/root/.cfy-agent下面，文件名是 [deamon.name].json.
  • workdir注意是当年目录，所以最好指定目录。
  • extra_env_path里面就是利用fabric打入的环境变量。

作用：

  • 这一步的作用就一个，创建daemon，并且将配置写进去。

3.配置代理

1

"/opt/cloudify_agent/deamon_name/env"/bin/python "/opt/cloudify_agent/deamon_name/env"/bin/cfy-agent daemons configure --name=deamon_name

作用

  • 生成pm/initd/initd.template模板的脚本，该脚本的作用是创建celeryd服务
  • 生成pm/initd/initd.conf.template模板的脚本，这个就是celeryd的配置
  • 生成broker配置文件

4.开始服务

1

"/opt/cloudify_agent/deamon_name/env"/bin/python "/opt/cloudify_agent/deamon_name/env"/bin/cfy-agent daemons start --name=deamon_name --interval=1 --timeout=120

作用：

  • 开始服务，说白了就是运行 sudo server [daemon.name] start

5.停止服务

1

"/opt/cloudify_agent/deamon_name/env"/bin/python "/opt/cloudify_agent/deamon_name/env"/bin/cfy-agent daemons stop --name=deamon_name --interval=1 --timeout=120

作用：

  • 停止服务，说白了就是运行 sudo server [daemon.name] stop

6.删除

1

"/opt/cloudify_agent/deamon_name/env"/bin/python "/opt/cloudify_agent/deamon_name/env"/bin/cfy-agent daemons delete --name=deamon_name

作用

  • 删掉配置文件。
